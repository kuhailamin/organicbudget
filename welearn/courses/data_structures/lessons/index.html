Data Structures and Algorithms in C++
  • A data structure is a particular memory arrangement of data that is suited for solving a particular problem. For example, stacks are based on the last-in-first-out principle (LIFO), and are suited for evaluating mathematical and undo/redo applications. As another example, queues are based on the first-in-first-out principle (FIFO), and are suited for providing access in a fair manner to a shared resource.
  • This course gives an overview of different linear and non-linear data structures. Examples of linear data structures include vectors, linked lists, queues, etc. Examples of non-linear data structures include trees and graphs. The course also introduces various algorithms for building and operating on the different data structures we will discuss in the course.

What are data structures used for?
  • They can be used for storing data, such as financial data, employee data, student data, etc. The emphasis here is on quick insertion, addition, search, and deletion.
  • They can be used for programmer's tools: Data structures such as hash tables, trees, and stacks can be used for keeping track of variables as well as evaluating and parsing programming statements.
  • They can be used for real-world modeling: We can model real-world situations with data structures. For instance, we can model airline routes or road maps with graphs. The graph nodes may, for instance, represent road intersections and the edges may represent the weights (e.g. distances).

What is an algorithm?
  • An algorithm is a step-by-step procedure that aims at solving a computational problem. Further, an algorithm typically has an input and an output. For instance, a sorting algorithm is an algorithm that solves the problem of sorting a set of numbers. The input is a set of unsorted numbers, and the output is the set of numbers being sorted.
  • Ideally an algorithm should be correct, readable, and efficient.
Algorithm vs. Function
  • What is the difference between an algorithm and a function?
  • A function is an implementation of an algorithm in a certain programming language such as C++.
  • An algorithm is an abstract representation of a computational procedure. It is written in pseudo code. Computer scientists use it so that they don't have to worry about specificities of a certain language. Software engineers can easily implement an algorithm in a specific language later on.
  • Unlike function code, algorithm code is not standardized. Hence, there may be many different ways to write the same algorithm. However, it is imperative that the algorithm code is easy to read.
  • Sometimes the terms "function" and "algorithm" are used interchangeably. Generally speaking, a function can also be considered an algorithm.
